This parameter allows you to control how XML schemas are sorted throughout the documentation.

It directly affects:

  1. The ordering of XML schema files in the Overview Frame (see also "Navigation | Overview Frame" parameter).
  2. In the case of framed documentation (generated by FramedDoc.tpl main template), the order, in which XML schemas become fully processed and documented (this includes the generation of Schema Overview and XSD Component Documentation pages for a schema). It determines which XML schema gets documented the first, and that, in turn, may affect the initial content of the frames:
    • Index Frame. (See parameter: "Navigation | Index Frame")
    • Detail Frame. (See parameter: "Navigation | Detail Frame")
  3. In the case of single file documentation (generated by PlainDoc.tpl main template), the order, in which the Schema Overview blocks (generated for each XML schema) follow each other.
The parameter has the following meaning:
  • When it is selected (true), the XML schemas will be sorted alphabetically by their file names (ignoring letter case). The XML schemas embedded in WSDL files are sorted first by the file name, then by the order number of the schema within the file.
  • When the parameter is unselected (false), the XML schemas will follow in a certain "physical" order determined by what WSDL/XSD files have been specified initially and how they reference (e.g. import) other WSDL/XSD files.
