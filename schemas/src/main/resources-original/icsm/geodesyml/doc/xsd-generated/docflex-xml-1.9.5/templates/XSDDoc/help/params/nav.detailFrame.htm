This parameter allows you to customize the initial content of the Detail Frame, which occupies the largest (right) part of the frameset:

(All the documentation informative content appears in this frame. Other frames are used mainly for navigation purpose.)

Possible Choices:

"Overview Summary"

Load the first the Overview Summary page.

The generation of this page is controlled by:

  • "Generate Details | Overview Summary" parameter
  • "Details | Schema Overview" parameter group
"All Component Summary"
Load the first the All Component Summary page.

The generation of this page is controlled by:

  • "Generate Details | All Component Summary" parameter
  • "Details | All Component Summary" parameter group
"Namespace Overview"
This selection will cause loading the first some of the Namespace Overview pages.

The documentation may contain several such pages -- a separate page for each namespace. Among them, the one will be loaded that was generated the first. This is controlled by the "Generate Details | Sorting | Namespaces" parameter.

The generation of Namespace Overview pages is controlled by:

  • "Generate Details | Namespace Overview" parameter
  • "Details | Namespace Overview" parameter group
"Schema Overview"
This selection will cause loading the first some of the Schema Overview pages.

The documentation may contain several such pages -- a separate page for each XML schema. Among such pages, the one will be loaded that was generated the first. This is controlled by the "Generate Details | Sorting | Schemas" parameter.

The generation of Schema Overview pages is controlled by:

  • "Generate Details | Schema Overview" parameter
  • "Details | Schema Overview" parameter group
Note:
When the specified page is not available (has not been generated), by default, the initially loaded document is chosen according to the following priority list:
  1. Overview Summary
  2. All Component Summary
  3. Namespace Overview
  4. Schema Overview
  5. Component Documentation
  6. Namespace Bindings Report
