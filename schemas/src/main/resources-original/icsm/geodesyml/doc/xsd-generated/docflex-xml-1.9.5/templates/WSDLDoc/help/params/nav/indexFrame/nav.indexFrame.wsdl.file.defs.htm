Specify whether to include the list of all top-level WSDL definitions found in the given WSDL file.

The definition list is broken into separate groups according to definition types (services, bindings, portTypes, messages). Using corresponding nested parameters, you can specify which types should be included in the list.

The sorting order of definitions within a type group is controlled by the nested parameter: "Sorting".

Default Value:

Copied dynamically from the value of the parameter: "Generate Details | WSDL Definitions"
