With these parameters you can filter which WSDL files and XML schemas should be documented in full details.

Precisely, WSDL file is considered any XML file containing <wsdl:definitions> as top element. XML schemas is identified by a <xs:schema> element found either as top XML file element or within <wsdl:definitions>/<wsdl:types> element (in that case, it is called embedded schema). Those things come from the following locations:

  • WSDL/XSD files specified initially as the generator arguments.
  • WSDL/XSD files directly or indirectly referenced from the initial files (via <wsdl:import>, <xs:import>, <xs:include>, <xs:redefine> elements).
Although all involved WSDL files and XML schemas must be loaded and processed, you may want actually to document only some of them. With these parameters you can specify which.

Note: The filtered out WSDL files and XML schemas (as well as the definitions/components found in them) may still be mentioned in the documentation, however with few details.
