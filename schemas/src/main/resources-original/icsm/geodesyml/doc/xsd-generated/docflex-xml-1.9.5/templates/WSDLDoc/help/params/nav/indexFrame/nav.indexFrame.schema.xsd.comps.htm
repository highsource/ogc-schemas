Specify whether to include the list of all global components and local elements defined in the given schema.

The component list is broken into separate groups according to component types (e.g. elements, complexTypes etc). Using corresponding nested parameters, you can specify which component types should be included in the list.

The sorting order of components within a type group is controlled by the nested parameter: "Sorting".

Default Value:

Copied dynamically from the value of the parameter: "Generate Details | XSD Components"
